# Activity

The **Activity** page is the course audit trail. Use it when you need to confirm what happened, who performed an action, or when a course item changed.

## Read the activity table

Each row can include:

- The person's name and profile image
- The action that was recorded
- A category, such as course, assignment, problem, submission, or grade activity
- The related assignment or problem
- A relative time and the full date and time
- An IP address, when one was recorded
- Additional metadata about the event

Select the information button in the **Metadata** column to see the related IDs and other recorded details. Some older events have less detail than newer ones.

## Find an event

Use the table search, sorting, and column filters to narrow the list. The category, assignment, and problem columns support filtering. Activity is shown newest first by default.

The first 50 events load when you open the page. Select **Load More** to retrieve older events. Select the refresh button near the **Activity** heading to check for new events.

## Common uses

The activity log is useful when you want to:

- Confirm that an assignment was published or edited
- Check whether a grade or roster change was recorded
- Trace activity related to a particular assignment or problem
- Investigate an unexpected course change before contacting an administrator

The log is a record of actions, not an undo tool. If an event shows a change that needs to be reversed, make the correction on the relevant course page.
