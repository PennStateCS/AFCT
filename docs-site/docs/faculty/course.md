# Course

The course page is the main workspace for Faculty and TAs. Open a course from the dashboard to manage its assignments, problem bank, roster, grades, groups, activity, and settings.

## What you see at the top

The course header shows the course code and name, semester, credits, and current status. A course is **Upcoming**, **Open**, or **Closed** based on its start and end dates.

The header also lists the assigned Faculty and TAs. Staff can copy either the registration code or a ready-to-share invite link. Students can use either one to join while self-registration is open.

## Course pages

| Page                          | What it is for                                                                       |
| ----------------------------- | ------------------------------------------------------------------------------------ |
| [Assignments](assignments.md) | Create assignments, set deadlines, add problems, and publish work to students.       |
| [Problems](problems.md)       | Build and maintain the reusable problem bank for the course.                         |
| [Roster](roster.md)           | Enroll students and review course membership. Faculty can also manage course roles.  |
| [Grades](grades.md)           | Review grades by student and assignment, then open a detailed problem breakdown.     |
| [Groups](groups.md)           | Create groups and manage group membership for group assignments.                     |
| [Activity](activity.md)       | Review actions recorded for the course.                                              |
| [Settings](settings.md)       | Update course details, dates, timezone, registration window, and publication status. |

## A good setup order

1. Open [Settings](settings.md) and confirm the course timezone, dates, and self-registration window.
2. Open [Roster](roster.md) and add any Faculty, TAs, or students who need access.
3. Create reusable items in [Problems](problems.md).
4. Create an [Assignment](assignments.md), add its problems, and confirm the points and submission limits.
5. Publish the course and assignment when students should be able to see them.
6. Use [Submissions](submissions.md) to review student work, leave comments, and assign grades.

## Published and archived courses

Publishing a course makes it visible to enrolled students. An unpublished course is still available to Faculty and TAs for setup.

AFCT will not let you unpublish a course after it has submissions or grades. This protects student work from being hidden accidentally.

Archived courses are read-only. Faculty and TAs can still review the course, but they cannot change its assignments, problems, roster, groups, or settings. Only a site administrator can archive or restore a course.
