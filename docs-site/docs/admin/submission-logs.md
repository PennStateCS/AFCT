# Submission Logs

The **Submission Logs** page shows submissions across the AFCT installation. It is useful when an administrator needs to investigate an evaluator problem that is not limited to one course.

## Choose the scope

The filters build on one another:

1. Select one or more courses.
2. Select assignments from those courses.
3. Select problems from those assignments.

Use **Select All** to select the available courses, assignments, and problems. Use **Clear Filters** to start over.

Status filters let you focus on the timing and evaluation results you need. The table shows the student, course, assignment, problem, submission date, grade, and status. Course, assignment, and problem names link back to the related review pages.

## Inspect a submission

The actions on each row let you:

- View evaluator feedback
- Open the submitted file in the correct AFCT viewer
- Download the original file
- Rerun the submission

A pending or processing submission cannot be rerun yet.

## Rerun visible submissions

The **Rerun** button at the top reruns every submission currently visible after the course, assignment, problem, and status filters are applied.

Start with the narrowest useful filter. A broad rerun can place avoidable work on the evaluator and may make it harder to isolate the original failure. After rerunning, check the updated status and feedback before changing grades manually.

For normal course grading and discussion, use the assignment's [Submissions](../faculty/submissions.md) page instead.
